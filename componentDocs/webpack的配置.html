    # 配置

    webpack 构建运行在 Node 环境下，所以文件需要通过 CommonJS 导出一个描述如何构建的 object 对象。

    通过从 main.js 出发，识别模块画导入语句，递归的寻找出入口文件的所有依赖，把入口和其他依赖打包到一个单独的文件中。从 webpack 2 开始，已经内置了对 ES6、CommonJS、AMD 模块化语句的支持。

    Webpack 可以把一切文件都可以看作是模块，通过引入 css 文件，但是 webpack 只支持 js、json。不原生支持解析 css 文件，要支持 非 js 类型的文件，需要使用 webpack loader 机制。

    # entry

    入口，构建的第一步将从 entry 开始。

    字符串，只能配置一个入口
module.exports = {
    entry: './src/main.js'
}
可以这些模块同时作为入口模块，会打包到一起
module.exports = {
    entry: ['./src/main.js', './src/index.js']
}
对象，可以配置多入口
module.exports = {
    entry: {
         main: './src/main.js'
    }
}
string、array 的 chunk 名字为 main。
object 是自己定义的名字.

    # Output

    输出结果。

    module.exports = {
    输出文件的名称
    filename: 'bundle.js' ,
    绝对路径
    path: path.resolve(__dirname, 'dist'),
    用来配置无入口的 chunk 在输出时的文件名称。
    chunkFilename: 'bundle.js',
    配置发布到线上资源的 URL 前缀。
    publicPath: '',
    用来配置库时候，需要用到。
    以何种方式导出。
    libraryTarget: '',
    导出库挂在 window 上的名称。
    library: ''
}

module.exports = {
    输出文件的名称
    filename: '[name].js' ,
}

占位符包括： 
id: chunk 的唯一标识，从 0 开始。
name: chunk 的名称。
hash: chunk 的唯一标识。
chunkhash: chunk 内容的 hash只。

    # plugin

    plugin 用来扩展 webpack 功能的，通过在构建流程里注入钩子实现。

    # module

    模块，在 webpack 里一切都是模块，一个模块对应着一个文件。

    module.exports = {
    module: {
        rules: []
    }
}

    # loader

    loader 可以看作具有文件转换功能的翻译员，通过在 modue.rules 数组配置一组规则，告诉 webpack 在遇到哪些文件使用哪些 loader 去加载和转换。

    # Chunk

    代码块，一个 Chunk 由多个模块组合而成，用于代码合并与分割。

    # Resolve

    Resolve 配置 webpack 如何寻找模块对应的文件。webpack 内置 js 模块化语法解析功能，默认采用模块化约定好的规则。也可以修改默认的规则。
